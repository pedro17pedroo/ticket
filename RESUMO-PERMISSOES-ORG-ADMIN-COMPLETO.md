## 🔧 Scripts Criados

### 1. Script de Adição de Permissões
**Arquivo**: `backend/src/scripts/add-complete-org-admin-permissions.js`

Este script:
- Cria todas as 93 permissões necessárias (se não existirem)
- Adiciona todas as permissões a todos os roles org-admin (14 roles)
- Mostra progresso detalhado da execução

**Como executar**:
```bash
cd backend
node src/scripts/add-complete-org-admin-permissions.js
```

## 📝 Notas Técnicas

### Permissões no JWT
As permissões são carregadas no token JWT durante o login através do middleware de autenticação em `backend/src/middleware/auth.js`.

### Verificação no Frontend
O frontend verifica as permissões usando o hook `usePermissions()` no componente `Sidebar.jsx` para mostrar/ocultar menus.

### Estrutura de Permissões
Cada permissão tem:
- `resource`: O recurso (ex: tickets, users, roles)
- `action`: A ação (ex: view, create, update, delete)
- `displayName`: Nome amigável
- `description`: Descrição da permissão
- `scope`: Escopo (organization, client, system)
- `category`: Categoria para agrupamento
- `isSystem`: Se é uma permissão do sistema
