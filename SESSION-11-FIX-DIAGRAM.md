## ğŸ”§ The Fix Explained

### What Changed?

**BEFORE**:
```javascript
include: [{
  model: Priority,
  as: 'priority',
  attributes: ['id', 'name', 'order']
}]
```
- Sequelize creates a JOIN to priorities table
- This triggers automatic association detection
- Sequelize tries to join workflows table too (because of incident_workflow_id FK)
- UUID vs INTEGER comparison fails

**AFTER**:
```javascript
attributes: ['id', 'name', 'shortDescription', 'priorityId']
```
- No nested include = No automatic JOINs
- Just fetch the priorityId as a simple attribute
- Frontend can fetch priority details separately if needed
- No type mismatch errors

### Why It Works?

1. **Simpler Query** - Fewer JOINs = Less complexity
2. **No Auto-Associations** - Sequelize doesn't try to be "smart"
3. **Type Safety** - No UUID vs INTEGER comparisons
4. **Better Performance** - Faster query execution
5. **Same Functionality** - priorityId is still available
